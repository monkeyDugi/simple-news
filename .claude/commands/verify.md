# /verify $ARGUMENTS

현재 구현한 기능을 어떻게 검증할지 체크리스트를 생성.

## 동작

1. `$ARGUMENTS` 파싱:
   - 비어있음 → TODO.md의 "현재 단계"를 대상으로
   - `네이버 스크래퍼` / `요약 파이프라인` 같은 기능명 → 해당 기능
   - 파일 경로 → 해당 파일의 API/함수

2. `qa-verifier` 서브에이전트에 위임 (또는 동등한 형식으로 답변):
   - 골든 패스 (정상 시나리오)
   - 엣지 케이스 (실패 시나리오)
   - curl 명령 예시
   - DB 쿼리 확인법
   - 실패 시 디버깅 힌트

3. 체크리스트는 **직접 돌려보고 체크**할 수 있도록 구체적 명령·입력값 포함.

## 예시

- `/verify` → 현재 단계 검증
- `/verify 네이버 스크래퍼`
- `/verify app/api/articles/route.ts`
- `/verify AI 요약 파이프라인`
